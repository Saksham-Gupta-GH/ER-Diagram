About this Viewer

ER Diagrams & Modules

This interactive tool visualizes the PostgreSQL database schema by grouping tables into logical business modules. You can view the structure of tables, search for specific columns, and examine primary keys (PK).

Important: Implicit vs Formal Relationships

The lines drawn between tables represent formal Foreign Key constraints exactly as they exist in the database.

If a table has an implicit relationship (e.g., a venueId column that lacks a strict database constraint), a line will not be drawn, and the table may appear in the "Isolated Tables" list. You can use the </> View Mermaid Code button to manually edit the code and add missing relationships for visualization.

Reading the Notation

The diagrams use standard Crow's Foot Notation:

  • ||--o{ : One-to-Many (e.g., One User has Many Bookings)
  • ||--|| : One-to-One
  • }o--o{ : Many-to-Many
